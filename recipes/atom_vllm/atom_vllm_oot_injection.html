ATOM OOT flow in vLLM

ATOM works as vLLM OOT Platform
vLLM keeps server, scheduler, workers, and runtime flow
ATOM provides model implementation, attention, and execution
Step 1

vLLM server starts

vLLM starts the backbone runtime for serving, scheduling, and worker orchestration.

vllm serve
Step 2

Register ATOMPlatform

vLLM discovers the ATOM plugin and registers the injected platform.

register_platform() -> ATOMPlatform
Step 3

Inject ATOM model

ATOM updates the vLLM model registry so supported architectures resolve to ATOM wrappers.

register_model() -> ModelRegistry override
Step 4

Provide attention backend

When the model runs, the injected ATOM platform provides the attention backend to vLLM.

ATOMPlatform .get_attn_backend_cls() -> AiterBackend or AiterMLABackend
Step 5

Start running model

vLLM continues to drive the framework, while the model execution path runs through ATOM.

ATOMForCausalLM ATOMMoEForCausalLM
